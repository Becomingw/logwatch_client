# Changelog

## [0.1.5] - 2026-03-20

### Fixed
- 修复 `lw` 监控命令时 `Ctrl+C` 可能无法完整中断整组子进程的问题

## [0.1.4b1] - 2026-03-16

### Added
- 客户端为每次执行自动生成全局唯一 UUID `task_id`，明确该 ID 永不复用
- 新增对服务端 `409` 冲突码的细分处理：`task_deleted`、`task_id_conflict`、`task_not_running`
- 任务被服务端删除后，客户端会停止后续上报并将本地队列转为归档状态
- 离线状态机新增低频探测模式，避免在极差网络环境下频繁重连

### Changed
- 断点续传、批量日志上传、心跳上报统一接入新的冲突响应分类逻辑
- 删除后的服务端 tombstone 语义正式成为客户端兼容协议的一部分
- 当任务结束事件因网络问题持续未成功上报时，客户端会回退发送离线完成邮件，而不必等待完全熔断

### Documentation
- 补充客户端与服务端关于 `task_id`、删除任务、tombstone、低频离线探测的对齐说明

## [0.1.3] - 2026-03-01

### Added
- 新增 `lw --setup` 命令用于交互式配置客户端
- 新增 `lw --health` 命令用于检查客户端健康状态
- 实现基于 SQLite 的本地持久化队列，提升日志上传可靠性
- 增强传输层状态管理，新增重试逻辑

### Changed
- 使用 requests 库替换 urllib 进行 HTTP 操作，提升性能和稳定性
- 改进任务通知邮件格式和内容

### Documentation
- 更新 README 布局和样式
- 完善文档结构
